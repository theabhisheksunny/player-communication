Key Design Patterns:
- Strategy Pattern: MessageSender abstraction with two implementations
- Observer Pattern: MessageHandler callback interface
- Dependency Injection: Players receive MessageSender via constructor
- Single Responsibility: Each class has one clear purpose

Requirements Mapping:
1. Create 2 players → Both Main classes create two Player instances
2. Initiator sends first → Player1 sends "Hello" to start
3. Concatenate + counter → Player.onMessageReceived() implementation
4. Stop at 10 messages → While loop in Main classes
5. Same process → SameProcessMain with InProcessMessageSender
6. Document responsibilities → Javadoc on every class
7. Different processes → DifferentProcessMain with Socket classes
